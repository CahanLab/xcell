# Changelog

All notable changes to xcell are documented in this file.
Format follows [Keep a Changelog](https://keepachangelog.com/).

## [Unreleased]

### Added
- **Figure builder — grid overlay, bivariate coloring, highlight-overlay toggle.** Three additions to the figure builder: (1) **Grid overlay** — a checkbox in the toolbar turns on an N×N grid in every panel for visual reference. Color and line width are configurable, applied to all panels. The grid is drawn into the exported PNG too. (2) **Bivariate coloring** — pick two gene sets (one as red, one as blue) plus a colormap (default / pinkgreen / orangepurple / custom) per panel. Mirrors the main Embedding view's bivariate option. (3) **Highlight overlay toggle** — a per-panel "show highlight layers" checkbox blends the dataset's current Highlight layers (configured in the Embedding tab's Genes panel) over the panel's coloring.
- **Figure builder — multi-panel publication-quality figures.** A new **Figure** tab next to Embedding/Heatmap. Click **Create figure** at the bottom of the Cells panel: if cells are selected, those are the figure's cell set; otherwise it falls back to the full dataset. Cell positions are snapshotted at creation, so subsequent filtering / deletion / analysis won't corrupt the figure. The builder shows a grid of panels (1×1 through 3×3 presets) — each panel renders the same cells colored by its own choice of single gene, gene set, or metadata column, with its own color scale (viridis / plasma / magma / inferno / cividis / coolwarm / blues / reds), expression transform (none / log1p), title, background, and border. **Point size and opacity are figure-level** (shared across panels) — the typical use case is comparing the same cells under different colorings, where varying point size between panels would distract; both controls live in the top toolbar. Pan/zoom is also shared so panels stay aligned. **Per-panel stacking**: when a panel colors by expression or a numeric metadata column, high-value cells render on top so bright markers aren't occluded by overlapping low-value cells. Export to PNG at 1×–4× DPI scale; output goes to your browser's default download directory with a timestamped filename (e.g. `figure_2026-05-19_1937.png`). Designed for the typical "compare gene A vs gene B side-by-side over the same tissue section" workflow, eliminating the need to save individual views and stitch them in another program.

### Changed
- **Center-panel tab renamed: "Scatter Plot" → "Embedding"** — more accurate label for what the view actually shows.
- **Highlight overlay panel: collapsible + more compact.** The whole section now collapses behind a chevron with an "N active" badge. Per-layer rows are denser: dropped the redundant `GENE`/`CELLS` kind label, moved the threshold-mode dropdown into the header row, combined the threshold value and intensity slider into a single row alongside the histogram. About ~30% less vertical space per layer.

### Fixed
- **Embedding view preserves zoom/pan across data refreshes.** Previously, any op that re-fetched the embedding (cell delete, filter, normalize) reset the camera to default because the viewState `useEffect` ran on every bounds change. Now the camera only re-centers when the user explicitly switches embeddings (e.g. spatial → UMAP); in-place data updates keep the user's current view. As a related fix, the cell-delete handler now also re-applies expression coloring (re-fetches with the new cell indices) so the active gene/gene-set coloring stays in place across delete.
- **Highlight overlay no longer scrambles after analyses that change cell count.** A `useHighlightSync` hook watches `schema.n_cells` and rebuilds layers when it changes: geneset layers auto-refetch their per-cell scores in place (preserving color, intensity, threshold mode; clamping `lo`/`hi` into the new range), and cellset layers (selection / category masks) are dropped with a toast since their masks were bound to old cell indices. Previously, ops like `filter_cells` and `delete_cells` would shift cell indices and the cached highlight values would point to the wrong cells — the user had to manually remove and re-add each layer.

### Fixed
- **Masked cells were silently included in line-association and gene-cluster analyses.** When a line had projections (the typical workflow), `Find Associated Genes` and the multi-line equivalent sent every projected cell to the backend regardless of the active cell mask. Same problem in `Cluster gene set`: `cellContext='selection'` didn't intersect with the mask, and `'all'` ignored the mask entirely. Now all three paths filter projected/selected indices through `activeCellMask` before sending; `'all' + mask` is converted to `'selection'` with the mask-derived indices. Diff exp and scanpy ops already honored the mask. Known gaps that remain (out of scope for this fix): **marker genes** and **heatmap** group by `obs_column` and don't pass a cell-index list, so honoring the mask there needs a new backend param.

### Changed
- **Highlight overlay now supports multiple layers, threshold gating, and cell-set sources.** The single-color, gradient-weighted overlay has been replaced with a stack of independent layers, each with its own color, intensity, and gating rule. Three ways to add a layer: (1) **Gene set** — pick a set, threshold via mode (`≥`, `≤`, between) + a draggable histogram cutoff (same UX as Select-by-Expression). Cells passing the threshold get full intensity; others get 0. (2) **Selection** — snapshot the current cell selection as a frozen mask. (3) **Category value** — pick a categorical `.obs` column + value; all cells with that value light up. The Cells panel also has a small color-dot button next to each category value for one-click highlighting. Layers stack in creation order; overlapping cells get color-mixed. The plot still draws highlighted cells on top via a stable sort by max layer weight. Replaces the previous single-overlay UI (one gene set, normalized gradient).

### Added
- **Arrow-key nudge in Quilt mode (transform phase)** — once you've lassoed a cell subset, press the arrow keys to translate the selection. Step is ~0.2% of the embedding's spatial range per press; **Shift+arrow** is 10× larger (matches Figma / Illustrator convention). Up/Down move along the data y-axis (math y-up). The handler is no-op when typing in inputs/textareas.

### Added
- **Combine spatial sections** — a new way to load 2+ spatial-transcriptomics h5ads as one dataset for cross-sample analysis (e.g. comparing the same tissue across two timepoints). Sections are laid out left-to-right along the spatial x-axis with a gap proportional to the mean section width (default 5%); y is unchanged so vertical alignment is preserved. Gene index is the intersection across inputs; a new `sample` categorical .obs column tags each cell with its source-file label. Per-file UMAPs/PCAs are dropped — run scanpy on the combined adata. New backend route `POST /api/combine_spatial` (body: `{files: [{file_path, label?}], slot, gap_fraction}`). New UI: a "Combine sections" mode toggle in the Load modal — click .h5ad files in the browser to build a list with editable per-file labels and an adjustable gap percentage. v1 supports `.h5ad` files only with `obsm['X_spatial']`; the route rejects others with a clear error.
- **File menu** — new "File" toolbar button (left of Select) consolidating Load, Combine spatial sections, and Export under one dropdown. The standalone Load and Export buttons are gone; everything file-related lives behind one click.

### Added
- **Highlight overlay** — a third color channel that blends a single user-picked color (default green `#22c55e`) over whatever the active color mode is (single gene, gene set, bivariate, or metadata), weighted per cell by an independent gene or gene-set score. Use case: demarcate epithelium in green using an epithelial gene set while keeping bivariate red/blue on cartilage/muscle scores. New section in the Gene Panel below "Bivariate Coloring": gene-set picker + color picker + intensity slider + Apply / Clear. The score uses the same per-gene normalization + aggregation pipeline already used by gene-set coloring (configurable in Display Settings). Selected and masked cells are not blended so their visual cues stay intact. Highlight does NOT change `colorMode` — it's a true overlay. **Highlighted cells render on top of the stack** via a stable secondary sort by highlight weight inside the deck.gl data array, so the highlight is never occluded by overlapping unhighlighted cells (important for dense plots like spatial transcriptomics).

### Changed
- **Drawing a line/shape no longer prompts for a name and does not exit Draw mode**. Lines are auto-named "Line N" (sequential by total drawn count) and added immediately to the Shapes panel. The user can keep drawing additional lines without re-clicking Draw — Escape exits draw mode. Lines can still be renamed in-place from the Shapes panel.
- **Adjust → Rotate is no longer Shift-gated and rotates around the data centroid**, not the viewport center. In Rotate mode, plain drag rotates; the cursor angle is measured relative to the data centroid (projected to screen), so cursor motion corresponds 1:1 to applied rotation. A faint orange dashed ring at the pivot and a live angle badge appear during the drag. Hold **Shift** to snap to 15° increments. The status row's "Flip X / Flip Y" buttons (Rotate mode only) are replaced with a rotation widget: ↺/↻ 90° quick buttons + a numeric input that accepts a precise degree value (Enter or Apply). Quilt mode's flip buttons are unchanged (they're contextual to the transform phase).
- **Adjust dropdown now has a third section: Flip → Flip Horizontal / Flip Vertical** as one-shot actions (no mode entry). Flip Horizontal mirrors left↔right (reflects about the y-axis); Flip Vertical mirrors top↔bottom (reflects about the x-axis). Tooltips include the math description for users who think in terms of axes. If Quilt has a current selection, the flip applies only to those cells; otherwise it flips the whole embedding around its centroid.

### Added
- **Categorical coloring now honors `adata.uns[f"{col}_colors"]`** (the scanpy/cellxgene convention). When a categorical `.obs` column has a pre-assigned color list, the scatter plot legend and per-cell fills use those exact hex values instead of the default palette. Loading an h5ad from scanpy / cellxgene / Seurat-via-SeuratDisk with stored cluster colors will now display those colors directly. Per-category colors are exposed on `GET /api/obs/{col}` (`colors: string[]`, positional) and `GET /api/obs/summary/{col}` (per-entry `color`). Label merges (`POST /api/obs/{col}/merge_labels`) rebuild the `_colors` list so palette state survives — the merged label inherits the existing fold-into target's color when folding into an existing label, otherwise the first merged label's color.

### Changed
- **Categorical palette is now length-aware**. When `adata.uns[f"{col}_colors"]` is absent, xcell still uses the d3-cat10 palette for the first 10 categories (existing aesthetics) and extends with golden-angle HSL hue stepping beyond that. Columns with >10 categories no longer wrap and produce duplicate colors — every category gets a unique color regardless of count.

### Fixed
- **Annotations created mid-session (Add Annotation in the Cells panel, Label cells from Select by expression) now appear in the Compare Cells column dropdown** — and in any other picker that reads `schema.obs_dtypes`, including the Heatmap config column picker. Previously, only `useObsSummaries` was refreshed after annotation creation, so the in-memory schema stayed stale until a page reload. Now `createAnnotation` flows re-fetch the schema via a shared `refreshSchema()` helper, the same way Scanpy operations already do.
- **Gene list import (JSON)** now accepts the [geneset-builder](https://github.com/) export schema (top-level object with a `sets` array) in addition to the legacy top-level array. It also tolerates per-gene objects of shape `{symbol, from}` (the raw `.gsb.json` project file shape), so a project file copied or renamed alongside a proper export still imports cleanly. Previously, any JSON whose top level wasn't an array failed with `No gene lists found in <file>`.
- **Marker genes (Compare Cells with 3+ groups) could return gene names that didn't exist in `.X`**, causing "Gene not found in dataset" when trying to color cells by them. Root cause: `run_marker_genes` called `scanpy.tl.rank_genes_groups` without `use_raw=False`. When the loaded h5ad had a populated `adata.raw` with a larger gene set than `adata.X` (e.g., a curated dataset where `.X` was trimmed but `.raw` kept the full transcriptome), scanpy's default `use_raw=True` silently pulled marker names from `.raw.var_names`. Those names were unreachable for visualization (which reads from `.X`), and worse, the marker list could include genes the user had explicitly removed via `exclude_genes`. Now uses `use_raw=False`, matching the two-group diff-exp path which was already correct. Existing saved marker gene sets still contain the stale names; re-run Compare Cells to get a clean set.
- Pass `activeSlot` explicitly to `runMarkerGenes`, `useDiffExp.runComparison`, and `ScanpyModal`'s Compare-Cells diff-exp call. (Defensive — `appendDataset` already auto-defaults to the store's `activeSlot`, but the explicit thread makes slot intent obvious at the call site.)
- Select → Lasso could silently stop drawing after switching between Lasso and Polygon. `handleMouseDown`'s `useCallback` deps array omitted `selectionTool`, so once the user had selected Polygon and then triggered any other dep change (toggling interactionMode, etc.), the handler stayed bound to a stale closure with `'polygon'` captured and returned early on every drag — leaving the user in "lasso mode" by the toolbar but with dragging doing nothing.
- Cell selection from Contourize annotations (and any other categorical `.obs` column with non-string category values) now works. Clicking a contour level in the Cells panel selects the corresponding cells; the same fix unblocks the "Compare Cells" picker in the Scanpy modal for these columns. The bug was a wire-format mismatch — `/api/obs/{col}` returned raw float categories while `/api/obs/summary/{col}` stringified them, so the frontend's `indexOf` lookup never matched.
- Segmented-line draw tool: the first segment is now drawn on the first try. Previously, selecting "Segmented Line" from the Draw dropdown after already being in draw mode left `handleMouseDown` / `handleMouseUp` bound to a stale `drawTool` closure (they didn't list `drawTool` as a `useCallback` dep), so the first click followed the pencil code path, cleared the line points, and the segmented line never accumulated points until the user toggled Draw off and on.
- Line association (clustering off): the results modal now renders the significant genes as collapsible Increasing/Decreasing cards, shows filter controls (min R², min amplitude, max FDR) for them, and enables "Add to Gene Sets" — previously the modal showed "No significant genes found" and the button was silently disabled because the old list-view path only handled module output.
- Spatial Autocorrelation no longer errors with `Expected n_perms to be positive, found 0` when Permutations is set to 0. The modal already documented 0 as the "analytical only" setting; the backend now translates `n_perms=0` to `None` (squidpy's analytical-only path) before dispatch.

### Changed
- **Display Settings → Gene Set Aggregation** is now an explicit pipeline rather than a single Z-score toggle. Pick a per-gene normalization (None / Z-score MAD / Z-score SD / Min-max / Rank → [0, 1]) and an aggregation (Mean / Median / Sum / Max); a per-gene clip slider appears when Min-max is selected. Same pipeline drives both single gene-set coloring and each axis of bivariate coloring (Contourize is unchanged). Defaults reproduce the previous Z-score+Mean behavior. The old `scoring_method` request param is gone — replaced by `per_gene_norm`, `per_gene_clip`, `aggregation`.
- Line association, "Add to Gene Sets" without clustering: now saves a single combined gene set named `Associated genes (N)` inside the per-run folder, instead of two separate Increasing / Decreasing sets. With clustering on, one set per module is still created.
- Overflow (`⋯`) menu popup on gene rows inside an expanded gene set is no longer clipped by the gene set container's rounded-corner mask — the popup now escapes ancestors with `overflow: hidden` and closes on scroll
- Stale cell state after `filter_cells` no longer corrupts downstream features: cell selection, expression coloring cache, and bivariate expression cache are all cleared when cells are removed, and the Select cells by expression modal defensively clamps any out-of-range indices before handing them to the backend
- Single-line gene association (gear icon in Shapes) now uses the line's projected/linked cells instead of the global cell mask, so only cells actually linked to the line are tested
- Fixed B-spline regression in line gene association: removed redundant intercept column that caused multicollinearity with the B-spline partition of unity, producing unreliable F-statistics and missing significant genes

### Changed
- **Combine Neighbors no longer overwrites the expression kNN by default**. The Analyze → Cell Analysis → Combine Neighbors modal now has an "Output graph name" input (defaulting to `combined_connectivities`); the result is written to `obsp[name]` instead of unconditionally clobbering `obsp["connectivities"]`. A `_connectivities` suffix is auto-added if missing so the new graph shows up in pickers (Smooth, future Combine runs). Picking the literal name `connectivities` keeps the old behaviour (overwrites the expression kNN and re-points Leiden / UMAP to the combined graph) and shows an inline yellow warning so users know what they're doing.

### Added
- **In-plot `Source:` layer dropdown for visualization**. A new dropdown appears in the bottom-left of the main scatter view (alongside the Embedding selector) whenever the active dataset has any non-X layer (typically after running Preprocess → Smooth). Picking a layer reroutes single-gene, gene-set, and bivariate coloring through that layer so you can visually inspect smoothed (or any other layered) values without modifying `.X`. The selection is per-dataset, persists for the session, and falls back to `.X (default)` if the chosen layer disappears (e.g. after reloading the dataset). Single-gene, multi-gene aggregation, and bivariate expression endpoints all gained an optional `layer` argument that overrides `transform` when set.
- **kNN smoothing in Preprocess → Smooth**. New Scanpy modal entry that smooths gene expression over any cell-cell kNN graph in `obsp` (Spatial Neighbors, Cell Neighbors, or a Combined graph). Algorithm: one or more iterations of `S ← D⁻¹·(A + α·I)·X` where α is the self-loop weight. The result is written to `adata.layers[output_layer]` (default `smoothed`) — `.X` is **not** modified. To use the smoothed values, pick the new layer from the `Source matrix` dropdown that now appears on Gene PCA, Gene Neighbors, and the Cluster Genes modal. New endpoints: `POST /api/scanpy/smooth`, `GET /api/scanpy/layers`. New optional `layer` parameter on `POST /api/scanpy/gene_pca`, `POST /api/scanpy/gene_neighbors`, and `POST /api/cluster_gene_set` (defaults preserve the historical `.X` behavior).
- **DBSCAN added as a third method in the Cluster genes modal**, alongside Hierarchical (Ward) and K-means. Picking DBSCAN replaces the K input with two new inputs: `eps` (neighbor radius in correlation-distance space, default 0.3 — i.e. genes are considered neighbors when their Pearson correlation is at least 0.7) and `min_samples` (minimum genes within `eps` for a gene to seed a cluster, default 3). Unlike K-based methods, DBSCAN discovers the number of clusters from the data; genes that fail the density threshold are returned as a trailing "noise" sub-cluster.
- **Display Settings defaults configurable via `~/.xcell/config.yaml`**. New `display:` section accepts `point_size`, `point_opacity`, `background_color`, `color_scale`, `bivariate_colormap`, `expression_transform`, `clip_percentile`, `gene_set_per_gene_norm`, `gene_set_per_gene_clip`, `gene_set_aggregation`. Defaults are applied to every dataset slot at startup and again on each fresh dataset load. Unknown keys are silently ignored. Sample updated in `docs/config.example.yaml`. Restart required to pick up edits.
- **Rename and merge category labels** in any categorical `.obs` column (Contourize results, Leiden, user annotations, original metadata). Double-click a label name in the Cells panel to rename it inline. Click the `⋯` menu on a column header → "Merge labels…" to open a small modal that lets you pick 2+ labels and choose the merged name. Renames refuse name collisions (the toast suggests using merge instead); merges fold into an existing target label if you pick its name. New endpoints: `POST /api/obs/{column}/rename_label`, `POST /api/obs/{column}/merge_labels`.
- **Color-ramp clip percentile slider** in Display Settings (0–5% in 0.5 steps, default 1.0). Controls symmetric percentile clipping for color-ramp anchors across single-gene, gene-set, and bivariate coloring. Set to 0 to disable clipping entirely. Single-gene expression now honors this setting too (previously it had no clipping).
- **Raw-counts snapshot on load**: when a dataset is loaded and `adata.X` looks like non-negative integer counts (detected from a bounded random sample), `adata.X` is copied into `adata.layers["counts"]` (only if that layer doesn't already exist). Default expression paths still read from `adata.X`; this is purely defensive so downstream tooling can reach raw counts after normalize/log1p is run in-session. Also sets `adata.uns["xcell"]["counts_inferred"] = True`.
- **User-editable config file for parameter defaults** at `~/.xcell/config.yaml` (or `.yml` / `.json`, or `$XCELL_CONFIG_PATH`). Override the default values for any Scanpy modal param (`scanpy.<function>.<param>`) or the Line Association form (`line_association.<param>`) without touching code. Missing file is fine — hardcoded defaults apply. Requires a backend restart to pick up edits. Sample at `docs/config.example.yaml`. New endpoint: `GET /api/config/defaults`.
- **Gene sets now survive accidental browser reloads**: the full gene-set category / folder / set tree is persisted on the backend process (server-lifetime, matches how drawn lines work) and rehydrated when the tab loads. The frontend debounce-saves on every mutation. Does not persist across backend restarts. Endpoints: `GET /api/gene_sets`, `PUT /api/gene_sets`.
- Line association results modal now has a **Download CSV** button that exports per-gene stats (`gene`, `f_stat`, `pval`, `fdr`, `r_squared`, `amplitude`, `direction`) for every gene tested — not just the top/significant ones. Suitable for running GSEA or other external ranked-list analyses. The backend response gains an `all_genes` field carrying the full table.
- Combine Neighbors (Analyze → Cell Analysis): weighted combination of multiple cell kNN graphs (e.g. expression neighbors + spatial neighbors). Each graph is max-normalized before weighting so graphs with different edge scales contribute proportionally to their weight, then summed and symmetrized. The result replaces `obsp['connectivities']`, so downstream Leiden/UMAP run on the combined graph with no further configuration. Default weights are equal across selected graphs and are user-editable.
- Line association: new "Cluster genes into modules" checkbox in the Gene Association section of Line Tools (single-line and multi-line). Default is **off** — only positive/negative gene lists are returned. Turning it on runs the existing profile-shape clustering (increasing, decreasing, peak, trough, complex).
- Line association: results now land in their own **Line Association** category in the Gene Panel, with each run of "Find Associated Genes" creating its own folder named after the line(s) and a timestamp. Gene sets inside the folder are one-per-module (clustering on) or one-per-direction (clustering off).
- PCA Loadings explorer (Analyze → Cell Analysis): scan a scrollable table of top +/− loading genes per PC, then create derived PC-subset embeddings (`X_pca_noPC2_5`, etc.) that exclude selected PCs. The Neighbors step gained a PC source dropdown that routes through any derived subset via `use_rep`; UMAP and Leiden inherit the choice automatically through the neighbors graph. Re-running PCA auto-clears stale derived subsets with a toast.
- Gene Mask: per-dataset, session-only filter that hides genes in the Gene Panel based on `.var` boolean columns (e.g. `highly_variable`, `spatially_variable`, `mt`). Opens from the Gene Panel `⋯` → `Gene mask…`. Supports per-column Off/Keep/Hide toggles with configurable Keep-combine mode (ANY or ALL). Filters gene browse, search, expanded gene set rows, and gene set score aggregation. Existing analysis operations (Diff Exp, Marker Genes, Gene PCA, etc.) are unaffected and retain their own gene_subset dropdowns.
- Select cells by expression threshold from gene and gene set `⋯` overflow menus. Interactive histogram with Above / Below / Between threshold modes; Update-selection action supports Replace / Add / Intersect; Label-cells action creates an annotation column with customizable high/low labels and context, and hands off directly to differential expression for the newly labeled groups.
- Overflow menu (`⋯`) on gene set and folder rows, consolidating Pin/Unpin and other secondary actions. On manual folder rows, the per-folder export moves into this menu as an `Export ▸` submenu.
- "Cluster genes…" action in the gene set overflow menu: sub-cluster a chosen gene set into K groups by expression pattern using hierarchical (Ward / correlation distance) or K-means. Supports running over all cells, the current lasso selection, or an annotation category. Results appear as a new folder under Gene Clusters.
- Manual gene set folders in the Gene Panel: create via a `+ 📁` button, rename via double-click, delete with a confirm dialog for non-empty folders. Sets can be created directly inside a folder or dragged in from the top level.
- Drag-reorder for gene sets (within the Manual category) and folders (in any category), with a teal insertion indicator above/below the drop target.
- Pinning for gene sets and folders: float an item to the top of its container via the `⋯` overflow menu. Pin state survives moves but is not exported.
- Hide/unhide for gene set categories: the 👁 button hides a whole category; a footer at the bottom of the Gene Panel lists hidden categories with per-category Unhide buttons.
- Per-folder export for manual folders to JSON, GMT, or CSV via the `⋯` overflow menu `Export ▸` submenu on the folder row.
- Support for loading prefixed 10x file trios (e.g. GSM1234_barcodes.tsv.gz, GSM1234_features.tsv.gz, GSM1234_matrix.mtx.gz) common in GEO accessions — detected automatically in the file browser and loadable like standard 10x matrix folders
- Header now shows the loaded data filename above cell/gene counts, with stacked stats layout
- GitHub icon link and Docs link in the header for quick access to the xcell repository
- Compare Cells feature in Analyze > Cell Analysis: select an .obs column, check 2+ groups, and run pairwise differential expression or marker gene analysis directly from the Analyze modal
- Lasso draw tool: draw closed freehand shapes in Draw mode (like pencil but always creates a closed shape)
- Polygon selection tool for Select mode: click vertices to define a polygon selection area, double-click to close and select cells inside
- Select button dropdown (replaces Lasso button): choose between Lasso (freehand) and Polygon (click-based) selection tools
- Adjust button dropdown: combines Rotate (formerly Adjust) and Quilt into a single toolbar dropdown

- Load modal redesigned with Finder-inspired two-column layout: sidebar with quick-access shortcuts (Home, Desktop, Documents, Downloads) and recently loaded files, breadcrumb path navigation for clicking any ancestor directory
- Gene Panel: more compact layout with reduced padding, Browse button moved to header next to Import, empty auto-generated categories (Gene Clusters, Similar Genes, etc.) hidden until populated
- Cell Panel: lasso selection is now cleared after running a lasso-based comparison (Set as Group 1/2 → Compare)

### Fixed
- Expression coloring errors (e.g., "Gene not found") no longer block the scatter plot. The error appears as a dismissable toast that auto-clears after 5 seconds, and the color mode resets so the app remains usable.
- When a dataset is loaded with no embeddings, the message now says to use Scanpy to create embeddings, instead of the misleading "No data loaded" message.
- Cell Panel now properly refreshes when switching between datasets or loading a new dataset
- Support for loading `.rds` files containing Seurat objects. Requires R and the Seurat/SeuratDisk R packages to be installed. RDS files are automatically converted to h5ad format on load.
- Load 10x Genomics Cell Ranger `.h5` files in addition to `.h5ad` files. The file browser now shows both formats, and the Load modal accepts either.
- Gene Neighbors basis selection: choose between gene PCA embeddings or raw expression as the basis for computing gene-gene neighbors. When using expression basis, optional gene subset filtering and z-score scaling are available. This lets you compare PCA-based and expression-based gene similarity results.
- Quilt mode undo: after rearranging cells in quilt mode, click the "Undo" button or press Ctrl/Cmd+Z to revert the last transform. Supports multiple levels of undo within a quilt session. The undo stack resets when exiting and re-entering quilt mode.
- Quilt mode: a new interaction mode for rearranging tissue pieces in spatial transcriptomics data. Click "Quilt" in the toolbar, lasso a group of cells, then drag to translate, shift+drag to rotate, or use flip buttons to reposition the selected patch. Press Escape to re-select, or Escape again to exit. Transforms persist in h5ad export.
- Side-by-side dual scatter plot view: click "Split" in the toolbar to view both primary and secondary datasets simultaneously. Click a plot to make it the active dataset. Each plot has independent pan/zoom, its own embedding selector, and its own legend. Interaction tools (lasso, draw, adjust) work on whichever plot you click. Gene and gene set coloring applies to both plots simultaneously.
- Embeddings created by Scanpy (PCA, UMAP) are now automatically selected for viewing, even when the dataset had no embeddings initially.
- Backend multi-dataset support: load multiple datasets into named slots (`primary`, `secondary`, etc.). All API endpoints accept an optional `?dataset=` query parameter. New `GET /datasets` and `DELETE /datasets/{slot}` endpoints for managing loaded datasets. Existing single-dataset usage is unchanged.
- Dataset slot selector in Load modal — load a second h5ad file into the Secondary slot without losing the Primary dataset.
- Dataset switcher dropdown in header toolbar (appears when two datasets are loaded) — switch between primary and secondary views seamlessly.
- Contourize spatial analysis: assign cells to spatial expression contour levels based on a gene set. Select genes in the Gene Panel, open Scanpy Modal > Spatial Analysis > Contourize, configure smoothing and contour levels, and run. Creates a new categorical obs column that can be used for coloring and downstream analysis.
- One-vs-rest marker gene analysis: click "Markers" on any categorical column in the Cell Panel to identify marker genes for each group using Wilcoxon rank-sum test. Select groups, set top N genes, and optionally apply fold-change and expression fraction filters. Results can be added as gene sets to the new "Marker Genes" category in the Gene Panel.
- Additive lasso selection: hold Shift while drawing multiple lasso shapes to select non-adjacent groups of cells without losing the previous selection.
- Embedding rotation and reflection controls ("Adjust" interaction mode): shift+drag to rotate, flip X/Y buttons. Transforms modify adata.obsm in place and persist in h5ad export.
- JSON import support in Import Gene Lists modal (`.json` files round-trip with export).
- Preferred embedding auto-selection on dataset load: spatial > umap > pca (case-insensitive substring match).

### Changed
- Loading a dataset no longer reloads the page; the store is updated in-place for seamless transitions.
- All frontend API calls now include `?dataset=` query parameter when targeting the secondary dataset slot.
- Replaced "Markers" button and "G1"/"G2" group buttons on categorical columns with checkboxes and a unified "Compare" button in the header toolbar. Check 2 categories for pairwise differential expression, or 3+ for one-vs-rest marker gene analysis. Checked categories are pre-selected in the Marker Genes modal.
- Renamed "Shapes" panel to "Lines" panel.
- Line tools (smoothing, gene association, projection embedding) moved from an inline section that obscured the panel into a dedicated modal, launched via a gear button on each line row.
- Lines panel is now collapsible (click the header to toggle).
- Cells (left) and Genes (right) panels are now horizontally collapsible. Collapsing either or both gives more space to the scatter plot.

### Fixed
- Gene set export now includes genes from all categories and folders (previously only exported the legacy flat list, which was often empty).
