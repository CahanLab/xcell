⏺ Here's how cell projection onto drawn lines works in the current
  implementation:

  Frontend Projection (store.ts:1007-1084)

  The projectSelectedCellsOntoLine function performs the projection entirely in
  the frontend:

  1. Prepare the Line

  const linePoints = line.smoothedPoints || line.points  // Use smoothed if 
  available

  2. Compute Arc-Length Along Line

  const cumulativeDistances: number[] = [0]
  for (let i = 1; i < linePoints.length; i++) {
    const dx = linePoints[i][0] - linePoints[i - 1][0]
    const dy = linePoints[i][1] - linePoints[i - 1][1]
    cumulativeDistances.push(cumulativeDistances[i - 1] + Math.sqrt(dx * dx + dy
   * dy))
  }
  const totalLength = cumulativeDistances[cumulativeDistances.length - 1]

  3. Project Each Cell onto Closest Segment

  For each selected cell at coordinates (cx, cy):

  for (let i = 0; i < linePoints.length - 1; i++) {
    const [x1, y1] = linePoints[i]
    const [x2, y2] = linePoints[i + 1]

    // Standard point-to-segment projection formula
    // t = dot(cell - p1, p2 - p1) / |p2 - p1|²
    let t = ((cx - x1) * dx + (cy - y1) * dy) / (segmentLength * segmentLength)
    t = Math.max(0, Math.min(1, t))  // Clamp to segment

    // Closest point on segment
    const projX = x1 + t * dx
    const projY = y1 + t * dy

    // Distance from cell to projection point
    const dist = Math.sqrt((cx - projX)² + (cy - projY)²)
  }

  4. Store Projection Result

  Each cell gets a CellProjection object:
  {
    cellIndex: number,           // Which cell
    positionOnLine: number,      // 0-1 normalized position along arc-length
    distanceToLine: number       // Perpendicular distance to line
  }

  Current Limitation

  The projection only operates on selectedCellIndices - cells selected via lasso
   tool. This is why your earlier test (lasso-selecting cells, projecting, then
  running association) gave spatially clustered cells with similar positions.

  The activeCellMask is now used for association testing, but not for the
  projection step itself. If you want to project all cells in a category (not
  just lasso-selected ones), the workflow would need modification to allow
  projecting masked cells directly.

  Would you like me to add an option to project all cells matching the active
  mask onto a line?

⏺ That makes sense. The projection embedding uses:
  - X = position along line (0 to 1)
  - Y = distance from line (unbounded, in embedding coordinate units)

  If cells are spread far from the line, the Y values could be in the hundreds
  while X spans only 0-1. In the scatter plot, this extreme aspect ratio
  compresses the X axis so much that the 0-1 variation becomes visually
  imperceptible.

  A few options to address this:

  1. Normalize Y to 0-1 as well - Scale distance values to match the position
  range
  2. Use percentile-based scaling - Map distances to percentiles (0-1)
  3. Log-transform Y - Compress large distances while preserving small distance
  detail

  Would you like me to modify the create_line_projection_embedding backend
  function to normalize or scale the distance values so both axes are
  comparable?
